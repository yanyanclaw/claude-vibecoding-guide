# 第二章：AI Agent vs 瀏覽器聊天 AI，搞懂你在跟誰說話

## 你以為你在跟同一種 AI 說話

上一章你體驗了 AI 鬼擋牆。跟 ChatGPT 來回十幾輪，每輪都修一個 bug 製造兩個新的，最後崩潰關掉瀏覽器。

但你有沒有想過一個問題：為什麼同樣是「AI 幫你寫程式」，有人用得痛不欲生，有人卻真的做出了上線的產品？

答案不是他們比你聰明，也不是他們用了什麼神奇的 prompt。

答案是：**他們用的根本不是同一種工具。**

你在瀏覽器裡跟 ChatGPT 聊天，和有人用 Claude Code 做出一個完整的 app，這兩件事情的差距，大概相當於你用 Word 寫了一份食譜，和你真的走進廚房煮了一頓飯。

食譜寫得再詳細，飯不會自己煮好。

## 第一種：瀏覽器聊天 AI

你最熟悉的那種。打開 ChatGPT 或 Gemini 的網頁，左邊打字，右邊回答。

它的運作方式很簡單：你丟一段文字進去，它根據你的文字生成一段回覆。就這樣。

你說「幫我寫一個導覽列的 HTML」，它給你一段 HTML。你把那段 HTML 複製，貼到你的檔案裡，存檔，打開瀏覽器看結果。如果不對，你回去跟它說哪裡不對，它再給你一段新的，你再複製，再貼上，再看結果。

這個過程有幾個致命的問題：

**它看不到你的東西。** 你的專案裡有 15 個檔案，CSS 寫在一個檔案、HTML 寫在另一個、JavaScript 又是一個。ChatGPT 看不到這些檔案。它只看得到你貼給它的那一段。所以當它回你一段「修正後的 CSS」，它根本不知道你的 HTML 長什麼樣，不知道你用了什麼 class name，不知道你的 JavaScript 有沒有跟那些 class 綁在一起。它是在瞎猜。

**它改不了你的東西。** 它給你一段程式碼，但它沒辦法幫你打開檔案、找到要改的位置、把舊的換成新的。這些全部是你自己手動做。你就是那個人肉 copy-paste 機器。

**它記不住你的東西。** 你跟它聊了 30 輪，它的記憶就像金魚，不是真的只有七秒，但也差不多了。對話越長，前面說過的東西它越記不住。所以你第 5 輪才交代過「字體要用 Noto Sans TC」，到第 15 輪它又給你預設字體，不是它故意的，是它真的忘了。

**它不知道改了之後會怎樣。** 它沒辦法跑你的程式碼。它給你一段 CSS，但它看不到那段 CSS 渲染出來的畫面長什麼樣。它沒辦法檢查「改了這行之後，那邊會不會炸掉」。它只能根據文字推理，而程式碼是一種「差一個字元就炸掉」的東西。

所以你上一章經歷的那場鬼擋牆，不是 AI 太笨。是你在跟一個看不到你的畫面、碰不到你的檔案、記不住你說過的話、也沒辦法檢查結果的傢伙合作。

換成人類，這就像你請一個室內設計師幫你改裝客廳，但他不能來你家，只能透過電話聽你描述，然後在電話裡口頭告訴你「你把沙發往左移 30 公分，然後把那個 IKEA 層架轉 90 度」。你照做了，結果沙發擋住了門。你再打電話，他再口頭指揮。每一輪都在瞎子摸象。

會出問題才是正常的。

## 第二種：AI Agent

現在想像另一種情境。

你打開終端機，輸入 `claude`，跟它說：「我要一個作品集網站，有導覽列、自我介紹區塊、作品卡片用網格排列、底部有聯繫表單。整體風格走深色系，字體用 Noto Sans TC。」

Claude Code 開始動。它自己建好資料夾、自己寫 HTML、CSS、JavaScript，自己把檔案存好。你打開瀏覽器看結果，一個完整的網頁已經在那裡了。

你看了一下，覺得作品卡片的間距太密。你說：「卡片之間的間距加大一點，大概 24px。」

它打開你的 CSS 檔案，找到那一行，把 16 改成 24。就改那一個數字。你重新整理瀏覽器，間距變了，其他東西一個字都沒動。

你覺得漸層方向不對，說：「背景漸層改成從左上到右下。」它找到漸層的定義，改了方向。其他一切完好。

沒有鬼擋牆。沒有修 A 壞 B。沒有 AI 跟你道歉五百次。

為什麼？因為 AI Agent 跟聊天 AI 有幾個根本性的不同：

**它看得到你所有的檔案。** Agent 不是在一個空白的聊天視窗裡瞎猜。它真的打開你的專案，讀了你的 HTML、你的 CSS、你的 JavaScript。它知道你的 class name 叫什麼、你的顏色變數定義在哪、哪個 CSS 規則會影響到哪個元素。

**它直接改你的檔案。** 你說「間距改成 24px」，它不是生一整段新的 CSS 讓你複製貼上。它找到那個特定的 CSS 屬性，把數字從 16 改成 24。就改那一個數字，其他幾百行程式碼一個字都不碰。

**它能看到結果。** 改完之後，它可以渲染畫面、看到結果。如果改壞了，它自己會發現，自己修好，不用你來回十五輪才發現字體又跑掉了。

**它有完整的上下文。** 你的整個專案就在它面前。它不需要你一輪一輪地提醒「字體要用 Noto Sans TC」，因為它看得到你的 CSS 檔案裡寫著 `font-family: 'Noto Sans TC'`。

回到那個室內設計師的比喻：Agent 就像設計師直接來到你家，看到你的客廳、量了尺寸、知道門在哪裡。你說「沙發往左一點」，他自己搬、自己量、自己確認搬完之後不會擋住門。

## 差在哪？一張表看完

| | 瀏覽器聊天 AI | AI Agent |
|------|-------------|----------|
| 代表工具 | ChatGPT、Gemini、Claude.ai | Bolt.new、Lovable、Cursor、Claude Code |
| 看得到你的檔案嗎 | 看不到，你貼什麼它才看什麼 | 看得到，整個專案都在它面前 |
| 能改你的檔案嗎 | 不能，只能給你程式碼讓你自己貼 | 能，直接幫你改 |
| 能看到結果嗎 | 不能，它不知道畫面長什麼樣 | 能，改完直接渲染 |
| 記憶 | 對話越長，前面的越容易忘 | 不靠記憶，專案檔案就是上下文 |
| 你的角色 | 人肉 copy-paste 機器 | 導演，看結果下指令 |

一句話總結：聊天 AI 是電話那頭的顧問，你聽完建議還得自己動手。Agent 是直接來你家幹活的人。

## 那 Artifacts 和 v0 算什麼？

你可能記得上一章提到的 Claude Artifacts 和 v0。它們不太好歸類，算是介於兩者之間的東西。

它們有聊天 AI 的外殼，你在左邊打字，右邊出結果。但它們又有一個 Agent 才有的能力：右邊會直接渲染出一個可以操作的畫面，不只是給你一段文字。

不過，它們的能力到此為止。它們能生出一個好看的前端介面，但沒有後端、沒有資料庫、資料重新整理就不見了。上一章說過，它們像精美的 PowerPoint，看起來很像真的，但不是真的產品。

把它們想成是「可以預覽的聊天 AI」。比純聊天進步很多，但還不是 Agent。

| 等級 | 工具 | 能力 |
|-----|------|------|
| 聊天 AI | ChatGPT、Gemini | 給你程式碼文字，自己想辦法 |
| 可預覽的聊天 AI | Claude Artifacts、v0 | 右邊直接長出畫面，但只有前端 |
| AI Agent（瀏覽器） | Bolt.new、Lovable | 前端 + 後端 + 資料庫 + 部署，整套搞定 |
| AI Agent（本地） | Claude Code、Cursor | 完整控制，但需要開發環境 |

為什麼要搞清楚這幾層？因為你用錯工具，就會花大量時間在不該花的地方。

你想做一個能登入、能存資料、能分享給別人用的 app，卻在 ChatGPT 的聊天框裡死磕了三天。那不是你的問題，是你選錯了工具。就像你想蓋一棟房子，卻只帶了一把螺絲起子。螺絲起子沒有壞，但它不是拿來蓋房子的。

## 非工程師該用哪個？

直接講結論：

**起步：Claude Artifacts。** 打開瀏覽器就能用，十分鐘做出一個可以互動的前端原型，展示給老闆或客戶看。這是你的第一步，零門檻。

**進階：Claude Code。** 當你發現 Artifacts 不夠用了，需要後端、資料庫、真正上線的產品，就該學 Claude Code。它需要裝一點東西（Node.js、終端機），但這本書會手把手帶你搞定。學會這個，你就擁有矽谷 vibe coder 同等級的武器。

**那 Bolt.new 和 Lovable 呢？** 你可能聽過這兩個工具，它們也是瀏覽器裡的 Agent，不需要開發環境就能做全端產品。聽起來很美好，但這本書不推薦你用它們當主力，原因有兩個：

第一，**貴而且封閉**。Bolt.new 和 Lovable 的付費方案每月 $20 到 $50 美金，而且有用量上限，超過就要加錢。更麻煩的是你的程式碼住在它們的平台上，要匯出才能拿到。哪天它們改定價或是倒了，你的專案就卡在裡面。Claude Pro 每月一樣是 $20 美金，但你同時拿到 Claude.ai、Artifacts、還有 Claude Code，程式碼就在你自己的電腦上，你隨時看得到、搬得走。同樣的價格，一邊是被平台綁住的套餐，一邊是完整的 AI 工程師助手。

第二，**天花板太低**。Bolt.new 和 Lovable 只能做它們平台支援的東西。你想串一個冷門的 API？想用一個它們沒內建的框架？想跑一個自訂的腳本？抱歉，不行。Claude Code 跑在你自己的機器上，你想裝什麼就裝什麼，想做什麼就做什麼。學一次，能力沒有天花板。

簡單講：Bolt.new 和 Lovable 是套餐，方便但有限。Claude Code 是你自己的廚房，要花五分鐘學開瓦斯爐，但之後什麼菜都能煮。

這本書的路徑是：先用 Artifacts 體驗「我也能做東西」的感覺，再帶你進入 Claude Code 拿到完整的開發能力。

## 常見誤解

**「我用 ChatGPT 很厲害了，這就是 vibe coding 吧？」**
不是。你在 ChatGPT 裡面跟 AI 聊天、請它幫你寫程式碼、然後自己複製貼上，這是「用 AI 輔助寫程式」，但不是 vibe coding。Vibe coding 的核心是你描述意圖、AI 直接動手做出來，中間沒有你手動搬磚的過程。

**「那 ChatGPT 就沒用了？」**
當然有用。拿來問概念、釐清想法、整理需求、寫文案、學新東西，ChatGPT 還是很強。但當你要「做出一個東西」的時候，你需要的是 Agent。就像 Google Maps 幫你找路很厲害，但它不會幫你開車。

**「我買了『最強提示詞大補帖』，是不是就能用 ChatGPT 做出 app？」**
不行。問題不在你的 prompt 寫得好不好，是工具的物種選錯了。你的提示詞再完美，ChatGPT 還是看不到你的檔案、改不了你的程式碼、跑不了你的專案。花幾千塊買提示詞模板，不如花十分鐘裝好 Claude Code。

**「Agent 工具這麼好，為什麼大家不全都用？」**
因為才剛出來。Bolt.new 和 Lovable 是 2025 年才爆發的，很多人連聽都沒聽過。而且人的慣性很強，「打開 ChatGPT 問一下」已經變成肌肉記憶了，要切換到一個全新的工作方式需要時間。你能看到這裡，代表你已經在大部分人前面了。

## 本章小結

- 瀏覽器聊天 AI（ChatGPT、Gemini）是顧問：你問它答，但動手的是你
- AI Agent（Claude Code、Bolt.new、Lovable）是動手做事的助手：你描述，它直接改
- 中間還有 Artifacts / v0：可以預覽畫面，但只有前端
- 你在第一章經歷的鬼擋牆，根本原因是聊天 AI 看不到你的檔案、改不了你的程式、也看不到結果
- 非工程師的 vibe coding 入口是 Agent 工具，不是 ChatGPT 聊天框

搞清楚了聊天 AI 和 Agent 的差別，你可能已經迫不及待想打開 Claude 試試了。

但在那之前，你得知道 vibe coding 這個東西到底有多瘋狂。它不是幾個工程師在車庫裡搞的小玩具，是一場矽谷正在發生的、十幾萬非工程師參與的運動。而這場運動裡發生的故事，會讓你對「不會寫程式的人能做什麼」徹底改觀。
